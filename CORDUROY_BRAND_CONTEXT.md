## 02. 核心にある「AIへの思想」

これがすべての起点。コピー・構成・説明・サービス設計のすべてにここが滲む。

### AIは「全部任せるもの」じゃない

- 「丸投げ・検索でしか使わない」状態を卒業することが出発点
- 全部任せると人もAIも弱くなる
- **一緒に考えるからこそ、AIの良さが活きる**

### AIは「難しく使わなくていい」

- 複雑なプロンプトは不要
- AIの特性を知り、シンプルな型で対話するだけで十分
- 「専門用語で詰め込む」より、腹落ちする理解を優先する

### AIは「人生を面白くする相棒になれる」

- 仕事の効率化だけじゃない
- 自分の価値観・判断軸・体調のクセまで教え込むと、言葉足らずでも補完してくれる存在になる
- 今日のモヤモヤを整理したり、まだ言葉になっていない夢を話したり、新しい挑戦の背中を押してもらったり
- **AIを「育てる」感覚。少しずつ自分を教えながら、一緒に育っていく**

### AIを、検索窓のままで終わらせない

- 「知っている」だけでは人は変わらない
- 学んで終わりにしない。使い続けることが大事
- 型を知り、実践し、仲間と続ける——この3つが揃ってはじめて変化が起きる

## 03. ターゲット像

### メインターゲット
- AIに興味はあるが「使い続けられていない」人
- 「検索ツールとしての利用に留まっており、回答の精度に不満」がある人
- 自分専用のAI（第二の自分）が欲しい人
- オフラインで、同じくAIを楽しむ仲間と学びたい人
- プログラミング未経験者から、実務でAIを使いたいビジネスパーソンまで

### コールテンが向かない人
- 「即効性のある魔法のツール」を求める人
- 作業をすべてAIに外注したい人
- 一人で黙々と学べばいい人（コミュニティの価値を求めない人）

## 05. 体験講座の設計思想（重要）

### 体験会の目的（3つ）
1. 「パッケージで得られる価値をとことん感じてもらう」
2. 「ちゃんと学んでいけるという自信を持って帰ってもらう」
3. 「わかりやすい、という体験をしてもらう」

### 当日の構成
```
パート①：AIでこんなことができる（AI基礎デモ）
  - 実際にChatGPTを触ってもらう
  - 「自分でもできそう」と思わせる、わかりやすさ重視

パート②：自分の情報を入れると変わる（第二の自分デモ）
  - 何も入れていないAI vs 自分情報を入れたAI の出力比較
  - 「こんなふうに使えるようになる」を体感

パート③：3ヶ月プログラムの価値を伝える
  - 受講すると何が変わるか
  - 内容・価格・次回スタート日
  - その場で申込OK（押し売りなし）
```

## 07. 糸口会（コミュニティ）の設計思想

### コミュニティ名の意味
「糸口（いとぐち）」——始まりの入口、糸を引き出す最初の一手。
コールテン（Corduroy）の縦糸からのイメージ。

### 3つの価値
1. **AIを楽しむ仲間とつながれる**
   — 「同じ熱量」ではなく「同じくAIを楽しむ」が正確。暑苦しくない。
2. **月1回、実践する場所**
   — 話すだけでなく、実際にAIを使って作業する。コールテンの講師がそばにいる。わからないところを質問しながら作業できる。
3. **他の人の質問からも学べる**
   — 「そういう使い方もあるんだ」「そのケースはこうすればいいのか」という、自分では気づかなかった切り口・発想が得られる場

### 作業会の形式
```
冒頭10分：近況と今日やること共有
60分：もくもく作業（AIを実際に使う）
クロージング10分：今日できたこと・詰まったこと共有
```
- 水曜夜（第4水曜）19:00-21:00 or 土曜朝（第3土曜）10:00-12:00
- どちらか月1回参加。会場定員が許す場合、人数が少ないうちは両方参加OK
- 場所：福岡市・赤坂駅から徒歩すぐ

### 避けるべき表現・言い換え

| NG | OK | 理由 |

| AI勉強会 | AI講座 | 表記統一 |
| 同じ熱量の仲間 | 同じくAIを楽しむ仲間 | 「熱量」が暑苦しい |
| AIに本気で取り組む | AIを楽しむ | 本気・頑張る系は圧力になる |
| 手を動かす場所 | 実践する場所 | ニュアンス整理 |
| マサヤ / 向 / 向さん | コールテンの講師 | 個人名を前面に出しすぎない |
| 草原さんオフィス（赤坂） | 赤坂駅から徒歩すぐ | 内部情報をぼかす |

| 無料体験会 | 体験講座 | サービスとしての位置付けを明確に |
| 絶対・必ず・〜すべき | — | 断定・押し付けになる |

### トーンの原則
- **共感が先、ベネフィットは後**——「わかるわかる」からスタート
- **余白を残す**——押し付けず、読む人が自分で選べる感覚を大事に
- **人間味がある**——「平均的なビジネス文章」にしない
- **フラットで自然体**——優しいが、芯はある
- **小さな気づき**を入れる——何か1つ「そうか」と思えるものを

## 09. デザイン・UI 指針

### カラーパレット

| 変数名 | 値 | 用途 |
|---|---|---|
| `--sea` | #16614e | メインカラー（深緑）。CTAボタン、アクセント |
| `--ink` | #1a1a2e | テキスト・見出し |
| `--accent` | #d4a04d | ゴールド。セクションラベル、バッジ |
| `--cream` | #faf9f7 | 背景基調 |
| `--mint` | #e8f5f0 | ライトグリーン。ホバー、カード背景 |
| `--text-light` | #6b7280 | サブテキスト |
| LINE green | #06C755 | LINEボタン専用 |

### フォント
- 見出し和文: CraftMincho（細いセリフ体。哲学的・上品な雰囲気）
- 本文和文: Noto Sans JP
- 数字・英字: Space Grotesk

### タイポグラフィ構造（ヒーロー）
```
.h1-pre  → 小さく・細い（前置き・問いかけ）
.h1-main → 大きく・太い（メインメッセージ。<mark>でグラデーション強調）
.h1-post → 中くらい・グラデーション（締め・余韻）
```

### UI原則
- **フェードアップアニメーション**（IntersectionObserver）でスクロールに合わせて要素を表示
- ラジアルグロー背景（ぼんやりした光彩）で奥行き感
- Unsplash背景画像＋CSSオーバーレイで「情緒的」なフルウィドスセクション
- カードは角丸32px、ボーダー `rgba(22,97,78,0.12)`、影 `0 8px 40px rgba(5,32,24,0.07)`
- LINEボタンは必ず `#06C755`、ピル型（border-radius: 99px）
