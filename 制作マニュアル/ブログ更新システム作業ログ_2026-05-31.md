## 背景

ブログ記事を下書きとしてため、公開候補を指定し、AIでブラッシュアップし、レビューを通して公開する流れを安全にしたいという要望から、既存の Astro ブログに公開前チェックの仕組みを追加した。

今回の目的は CMS 化ではなく、現在の構成を壊さずに「下書きが検索エンジンや一覧に混ざる事故」「AI生成残骸が残る事故」「記事・一覧・sitemap の同期漏れ」を機械的に検出すること。

## 追加したもの

### 1. ブログ監査スクリプト

追加ファイル:

- `Public/scripts/blog-audit.mjs`

追加コマンド:

```bash
npm run blog:audit
npm run blog:audit:strict
```

主な検出内容:

- 一覧にあるのに記事ファイルが存在しない
- 一覧にある記事が `isPublished = false`
- `isPublished = false` なのに `noindex` がない
- `noindex` 記事が sitemap 除外に入っていない
- AI生成残骸の `</content>` / `</invoke>` が残っている
- 記事日付と一覧日付の不一致
- datePublished / dateModified の不足
- 内部ブログリンク不足
- llms.txt 登録不足
- TODO / REVIEW 系マーカー

### 2. 簡易実行スクリプト

追加ファイル:

- `Public/scripts/blog-check.sh`

追加コマンド:

```bash
npm run blog:check
npm run blog:check:quick
```

用途:

- `npm run blog:check`: strict 監査、build、typecheck をまとめて実行
- `npm run blog:check:quick`: strict 監査だけを素早く実行

今後の公開前レビューでは、基本的に `npm run blog:check` を使えばよい。
